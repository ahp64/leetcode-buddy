👋 Set up your streak

This dashboard has no group configured yet. Everything — members, timezone, reminders — is set with one secret on your repo:

  1. Open Settings → Secrets and variables → Actions. On the Repository secrets tab (not Environment secrets — that's a separate tab for the auto-created github-pages environment) add or edit a secret named BUDDY_CONFIG.
  2. Paste in a value shaped like this, with real LeetCode usernames — add a member per person: 
    {
  "settings": { "timezone": "America/Chicago", "reminderHours": [12, 18, 21] },
  "members": [
    { "name": "You", "leetcodeUsername": "your-leetcode-username",
      "email": "you@example.com", "notifyEmail": true },
    { "name": "Buddy", "leetcodeUsername": "their-leetcode-username",
      "email": "buddy@example.com", "notifyEmail": true }
  ]
}
  3. Run the dashboard workflow once to pick it up: Actions → Streak check & dashboard → Run workflow (it also runs itself automatically every 15 minutes).

⚡ Want these to work in one click right here? Connect GitHub

Create a fine-grained personal access token: under Repository access choose Only select repositories → this repo (not “Public repositories” — that forces read-only). Then in the Repository permissions list, set three rows to Read and write: Contents (commits freeze/unfreeze and, if you use it, the member list, directly — so changes apply instantly), Actions (nudges a faster public refresh after each change), and Secrets (lets this page update BUDDY_CONFIG and, if you fill them in below, your email/text delivery credentials). Leave every other row on “No access” (Metadata: Read-only adds itself — that's normal). Generate, copy the github_pat_…, paste it here. It is stored only in this browser — for one-click controls on your buddy's device too, they'll need their own token pasted in there (or you can share yours).

⚠️ This combination is broad: Contents can change any non-workflow file in the repo, and Secrets can silently overwrite any secret — including your SMTP/Twilio credentials, which control where reminders actually get sent (GitHub never lets a token, or anyone, read a secret's existing value back, so this is write-only exposure — a leak can't reveal past PII). Only paste this into a browser/device you trust; skip it entirely to keep using the deep-link buttons and Settings page instead, which need no token at all. This page also loads a small open-source encryption library (libsodium) from a CDN the first time it needs to write a secret.
